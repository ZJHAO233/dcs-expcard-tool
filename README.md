# DCS Converter 转换器说明

## 概述

DCS Converter 是一个将 Excel 试验条件表转换为 Markdown 文档的工具。

- **输入**: Excel 文件 (.xlsx)
- **输出**: Markdown 文件 (output.md)
- **核心类**: `DCSConverter` (convert.py)

---

## Excel 列结构映射

| Excel列 | 变量名 | 用途 |
|---------|--------|------|
| A列 | a | 编号/标题 (一、/1.1测试项/1/1.1) |
| B列 | b | 逻辑分隔符或内容 |
| C列 | c | 逻辑分隔符或内容 |
| D列 | d | 逻辑分隔符或内容 |
| E列 | e | 内容 |

---

## 逻辑运算符字典

```python
LOGIC_OPERATORS = {
    '与': '且',      # Excel中的"与"，输出为"且"
    '且': '且',      # Excel中的"且"，输出为"且"
    '或': '或',      # Excel中的"或"，输出为"或"
    '或取反': '或取反',  # Excel中的"或取反"，输出为"或取反"
}
```

## 程序执行流程

```
convert(file_path)
├── 读取Excel文件
├── 遍历每一行 _process_row(row, rows, i)
│   │
│   ├── 空行/表头 → 跳过
│   │
│   ├── 二级标题（一、xxx）
│   │   ├── 输出空行
│   │   ├── 输出 '## xxx'
│   │   └── 重置状态
│   │
│   ├── 段落标题（试验条件/试验恢复/结论/存在问题）
│   │   ├── 试验条件 → skip_section = True
│   │   └── 其他 → in_content_area = False
│   │
│   ├── 试验内容 → in_content_area = True
│   │
│   ├── 三级标题（数字.中文）
│   │   ├── 统一括号为全角
│   │   ├── 输出空行
│   │   ├── 输出 '### xxx'
│   │   ├── 输出空行
│   │   └── 记录 current_sub_title_logic
│   │
│   ├── 子列一级（纯数字）_process_level1()
│   │   └── 详见下方「子列一级处理」
│   │
│   └── 子列二级（x.1, x.2...）_process_level2()
│       └── 详见下方「子列二级处理」
│
└── 返回 '\n'.join(output)
```

## 子列一级处理 (_process_level1)

### 功能说明

处理子列一级行（如 `1`, `2`, `3`），输出格式为 `数字. 内容（逻辑）`。

### 处理流程

```
_process_level1(a, b, c, row, all_rows, index)
│
├── 前向查看下一行
│   ├── has_children = 下一行是否是 "a.x" 格式（如 "1.1"）
│   └── child_logic = 下一行B列是否是逻辑分隔符
│
├── 获取内容
│   └── content = 从B列开始找（跳过逻辑分隔符）
│
├── 判断是否输出逻辑标注
│   │
│   ├── 情况1: 有子项 && child_logic存在 && B列是逻辑分隔符
│   │   ├── B列逻辑 ≠ 三级标题逻辑 → 输出（B列逻辑）
│   │   └── B列逻辑 = 三级标题逻辑 → 忽略
│   │
│   ├── 情况2: 有子项 && child_logic存在 && B列不是逻辑分隔符
│   │   └── 输出（child_logic）
│   │
│   ├── 情况3: 无子项
│   │   └── 不输出逻辑标注
│   │
│   └── 特殊处理: '或取反' in B列
│       └── 输出（或取反）
│
└── 输出: "数字. 内容（逻辑）"
```

### child_logic 说明

`child_logic` 是从第一个子项的B列提取的逻辑分隔符。当子列一级的B列不是逻辑分隔符时，使用子项的B列逻辑作为标注。

### 示例

```
Excel数据:
1, A磨润滑油条件满足     → B列是内容
1.1, 与, 或, 与, www    → B列是"与"

处理过程:
1. 子列一级 "1"，B列="A磨润滑油条件满足"（不是逻辑分隔符）
2. 查看子项 "1.1"，B列="与"（是逻辑分隔符）
3. child_logic = "与"
4. 输出: "1. A磨润滑油条件满足（与）"
```

## 子列二级处理 (_process_level2)

### 功能说明

处理子列二级行（如 `1.1`, `1.2`），根据列的组合情况选择不同的处理方式。

### 列含义说明

```
Excel列:  A    B         C         D         E
含义:    编号  逻辑/内容  逻辑/内容  逻辑/内容  内容

判断顺序:
1. B列是逻辑分隔符? → 传递给父级
2. C列是逻辑分隔符?
   ├── D列也是逻辑分隔符? → 三级条件处理 (从E列找内容)
   └── D列不是逻辑分隔符? → 二级分组 (从D列找内容)
3. 简单输出 (从B列找内容)
```

### 处理流程

```
_process_level2(a, b, c, row, all_rows, index)
│
├── 获取各列值
│   ├── b_val, c_val, d_val, e_val
│   └── b_is_logic, c_is_logic, d_is_logic
│
├── 情况1: C列是逻辑分隔符 AND D列是逻辑分隔符
│   └── → 三级条件处理 (_process_level3_group)
│
├── 情况2: C列是逻辑分隔符 AND D列不是逻辑分隔符
│   └── → 二级分组 (_process_level2_group)
│
├── 情况3: B列是逻辑分隔符
│   ├── C列有内容(不是逻辑分隔符) → 输出C列内容
│   └── 其他 → 跳过
│
└── 情况4: 简单输出
    └── 从B列开始找内容 → "   - 内容"
```

### 情况详解

#### 情况1: 三级条件处理

当C列和D列都是逻辑分隔符时触发，从E列找内容。

```
触发条件:
├── C列是逻辑分隔符（如"或"）
└── D列是逻辑分隔符（如"与"）

处理逻辑:
├── 初始化分组: current_group_items = [E列内容]
├── 扫描后续行:
│   ├── D列有逻辑 → 新三级分组
│   ├── D列有内容(不是逻辑) → 单独项目
│   └── D列为空 → 从E列获取内容加入当前组
├── 构建输出: （内容1 且/或 内容2） 或 （内容3 且/或 内容4）
└── 结束条件: C列有内容但不是逻辑分隔符
```

#### 情况2: 二级分组

当C列是逻辑分隔符但D列不是时触发，从D列找内容。

```
触发条件:
├── C列是逻辑分隔符（如"或"）
└── D列不是逻辑分隔符（是内容或空）

处理逻辑:
├── 初始化分组: group_items = [D列内容]
├── 扫描后续行:
│   ├── C列有逻辑 → 新分组
│   ├── D列有逻辑 → 结束当前分组
│   └── 从C列找内容加入分组
├── 构建输出: 内容1 或/且 内容2
└── 结束条件: C列或D列有逻辑分隔符
```

#### 情况3: B列逻辑传递

当B列是逻辑分隔符时，逻辑传递给父级，当前行可能输出内容或跳过。

```
触发条件:
└── B列是逻辑分隔符（如"与"、"或"）

处理逻辑:
├── C列有内容(不是逻辑分隔符) → 输出C列内容
└── 其他 → 跳过（逻辑已传递给父级）
```

#### 情况4: 简单输出

当上述情况都不满足时，从B列开始找内容直接输出。

```
触发条件:
├── B列不是逻辑分隔符
├── C列不是逻辑分隔符
└── D列不是逻辑分隔符

处理逻辑:
└── 从B列开始找内容 → "   - 内容"
```

## 三级条件处理详解 (_process_level3_group)

### 功能说明

处理三级条件分组，当C列和D列都是逻辑分隔符时触发。

### 处理流程

```
_process_level3_group(a, c_logic, d_logic, first_content, row, all_rows, index)
│
├── 初始化
│   ├── level3_groups = []  ← 存储所有三级条件组
│   ├── current_group_items = [first_content]  ← 当前组的内容
│   └── current_d_logic = d_logic  ← 当前组使用的逻辑
│
├── 扫描后续行 (j = index+1)
│   │
│   ├── 循环: j < len(all_rows)
│   │   │
│   │   ├── 下一行是同级子项 (x.y格式)?
│   │   │   │
│   │   │   ├── C列有逻辑 → break (新二级分组)
│   │   │   │
│   │   │   ├── D列有逻辑 → 新三级分组
│   │   │   │   ├── 保存当前组: level3_groups.append((current_d_logic, items))
│   │   │   │   ├── 重置: current_group_items = [], current_d_logic = D列逻辑
│   │   │   │   └── 从E列获取内容加入新组
│   │   │   │
│   │   │   ├── D列有内容(不是逻辑) → 单独项目
│   │   │   │   ├── 保存当前组
│   │   │   │   ├── level3_groups.append((None, [D列内容]))
│   │   │   │   └── 重置: current_group_items = []
│   │   │   │
│   │   │   └── D列为空 → 从E列获取内容加入当前组
│   │   │
│   │   ├── 下一行是延续行 (A列为空)?
│   │   │   ├── 同上逻辑
│   │   │
│   │   └── 其他 → break
│   │
│   └── 循环结束
│
├── 保存最后一个分组
│
└── 构建输出
    ├── 遍历 level3_groups
    │   ├── logic=None → 直接输出内容
    │   └── logic=有值 → 用logic拼接, 多项用（）包裹
    └── 用"或"连接各部分 → "   - 结果"
```

### 关键点

1. **触发条件**: C列和D列都是逻辑分隔符
2. **内容来源**: E列
3. **分组逻辑**: D列决定如何拼接同一组的内容
4. **分组边界**: 遇到新的D列逻辑分隔符时，开始新组
5. **单独项目**: D列有内容但不是逻辑分隔符时，作为单独项目
6. **结束条件**: 遇到C列有内容但不是逻辑分隔符时，结束扫描
7. **输出格式**: 用`（）`包裹每组，用"或"连接各组

### 完整示例

```
Excel数据:
行21: 1.1, B=与, C=或, D=与, E=www
行22: 1.2, B=, C=, D=, E=eee
行23: 1.3, B=, C=, D=, E=rrr
行24: 1.4, B=, C=, D=与, E=ttt
行25: 1.5, B=, C=, D=, E=yyy
行26: 1.6, B=, C=, D=uuu, E=
行27: 1.7, B=, C=iii, D=, E=

处理过程:
1. 1.1: C=或, D=与 → 触发三级条件
2. 1.2-1.3: D列空 → 加入当前组 [www, eee, rrr]
3. 1.4: D=与 → 新组 [ttt]
4. 1.5: D列空 → 加入当前组 [ttt, yyy]
5. 1.6: D=uuu → 单独项目
6. 1.7: C=iii → 结束扫描

输出: （www 且 eee 且 rrr） 或 （ttt 且 yyy） 或 uuu
```

## 二级分组处理详解 (_process_level2_group)

### 功能说明

处理二级分组，当C列是逻辑分隔符但D列不是时触发。

### 处理流程

```
_process_level2_group(a, c_logic, first_content, row, all_rows, index)
│
├── 初始化
│   ├── group_items = [first_content]  ← 分组内容
│   └── continuation_logic = c_logic  ← 分组逻辑
│
├── 扫描后续行
│   │
│   ├── 同级子项?
│   │   ├── C列有逻辑 → break
│   │   ├── D列有逻辑 → break
│   │   └── 从C列找内容加入分组
│   │
│   ├── 延续行?
│   │   └── 同上
│   │
│   └── 其他 → break
│
└── 输出
    ├── 多项 → "内容1 + logic分隔符 + 内容2"
    └── 单项 → "内容"
```

### 完整示例

```
Excel数据:
行31: 3, sss
行32: 3.1, B=或, C=ddd
行33: 3.2, C=与, D=fff
行34: 3.3, D=ggg

处理过程:
1. 3.1: C=或(逻辑), D=ddd(内容) → 触发二级分组
2. 初始化: group_items = [ddd], continuation_logic = "或"
3. 3.2: C=与(逻辑) → break
4. 输出: "   - ddd"

注意: 3.2的C=与是逻辑分隔符，所以结束当前分组
```

## Markdown 输出格式

| 行类型 | 输出格式 | 示例 |
|--------|----------|------|
| 二级标题 | `## 标题` | `## 一、A磨煤机` |
| 三级标题 | `### 标题` | `### 1.启动允许条件（与）` |
| 子列一级 | `编号. 内容（逻辑）` | `1. A磨润滑油条件满足（与）` |
| 子列二级(简单) | `   - 内容` | `   - eeeee` |
| 子列二级(二级分组) | `   - 项1 且/或 项2` | `   - aaaaa 且 bbbbb` |
| 子列二级(三级条件) | `   - （项1 且/或 项2） 或 （项3 且/或 项4）` | `   - （www 且 eee） 或 （rrr 且 ttt）` |

## 状态变量说明

| 变量 | 类型 | 作用 | 重置时机 |
|------|------|------|----------|
| `output` | list | 存储输出行 | convert() 开始时 |
| `in_content_area` | bool | 标记是否在"试验内容"区域 | 遇到二级标题/段落标题时重置 |
| `skip_section` | bool | 控制是否跳过当前段落 | 遇到"试验内容"或二级标题时重置 |
| `current_sub_title_logic` | str/None | 当前三级标题的逻辑关系 | 遇到新三级标题时更新 |
| `processed_rows` | set | 已处理行索引(防重复) | convert() 开始时 |
